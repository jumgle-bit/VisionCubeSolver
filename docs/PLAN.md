# VisionCubeSolver 实施计划

## 1. 目标与约定

VisionCubeSolver 是一个离线 Python 桌面应用。它通过电脑摄像头采集
3x3 魔方六面，允许用户修正识别结果，校验魔方状态，并使用两阶段
IDA* 与模式数据库生成求解步骤。

应用统一使用标准配色和固定观察方向：

- `U = White`
- `R = Red`
- `F = Green`
- `D = Yellow`
- `L = Orange`
- `B = Blue`
- 展示和解释步骤时始终为黄面朝下、绿面朝向用户

首版仅支持标准配色的 3x3 魔方。求解器优先保证实用速度，不保证全局
最短解。

## 2. 用户流程

1. 启动应用并选择摄像头。
2. 将一面放入画面中央的 3x3 采样框。
3. 选择当前面的中心块颜色。
4. 按照界面提示调整当前面的顶部方向。
5. 点击“确认当前面”，查看识别出的色块和置信度。
6. 手动修正误识别色块，并依次完成六面。
7. 应用校验颜色数量、块组合、方向和排列奇偶性。
8. 校验通过后点击“求解”，查看旋转步骤。

拍摄方向约定：

| 当前面 | 画面顶部参考颜色 |
| --- | --- |
| Green / Red / Blue / Orange | White |
| White | Blue |
| Yellow | Green |

## 3. 模块设计

### 3.1 摄像头与识别

- `CameraService`：枚举、打开、读取和关闭摄像头。
- `GridSampler`：从引导框内提取九个色块样本。
- `ColorClassifier`：使用 Lab/HSV 中位数、默认参考色和中心块校准色分类。
- `FaceRecognitionResult`：保存九个颜色、置信度和原始样本。

识别前期使用默认 Lab 参考色。收集中心块后使用中心块样本更新参考值；
六个中心块齐全时重新识别已采集的所有面。

### 3.2 魔方模型与校验

- `FaceletCube`：保存 `URFDLB` 顺序的 54 个面块。
- `CubieCube`：保存 8 个角块与 12 个棱块的排列和方向。
- `CubeSession`：管理六面采集、手动修改、校验和求解输入。

校验规则：

- 六个中心块必须符合固定标准配色。
- 每种颜色必须恰好出现九次。
- 所有角块和棱块必须唯一且组合合法。
- 棱块翻转总和必须为偶数。
- 角块扭转总和必须模 3 为零。
- 角块排列与棱块排列奇偶性必须一致。

### 3.3 两阶段 IDA* 求解

第一阶段把状态转换到子群：

- 所有角块方向归零。
- 所有棱块方向归零。
- 四个中层棱块进入中层集合。

第二阶段恢复全部排列，只允许：

`U U2 U' D D2 D' R2 L2 F2 B2`

模式数据库由坐标移动表和剪枝表组成，首次使用时生成并缓存。缓存带版本
信息和校验，文件不存在、损坏或版本不一致时自动重建。搜索禁止连续转动
同一面，并裁剪同轴冗余动作。

## 6. 验收标准

- `python -m vision_cube_solver` 能启动桌面应用。
- 用户能以任意顺序采集六面并手动修改。
- 非法状态显示具体错误且无法求解。
- 合法状态能在 30 秒默认超时内返回可执行解。
- 输出步骤以黄底、绿朝用户的固定方向解释。
- 摄像头不可用、低置信度、缓存损坏和搜索超时均显示明确提示。
- 单元测试覆盖识别、状态校验、动作和求解流程。
